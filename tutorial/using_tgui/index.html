TGUI

In this tutorial, I’ll share some information needed to use the TGUI library with modern CMake. This guide is very similar to the one for TGUI.

Installing TGUI

To make use of TGUI in your projects you first need to install it. For that i highly recommend compiling it yourself.

Downloading the TGUI source code

To compile it yourself, the source code is required. You can either use git like this:

git clone https://github.com/texus/TGUI.git

or download an archive from here and then extract it somewhere you like.

Building and installing TGUI

After downloading the source code and entering it’s directory, build it using CMake. I booth build and install debug and release binaries. Also i use the ninja generator, if you don’t just omit the -GNinja. Note that the SMFL_DIR variable needs to be set to the SFML installation directory when configuring. In my case thats “C:\Program Files (x86)\SFML”.

mkdir build
cd build
cmake .. -GNinja -DCMAKE_BUILD_TYPE=Debug -DSFML_DIR="</path/to/SMFL>"
cmake --build . --target install -- -j 4
cmake .. -GNinja -DCMAKE_BUILD_TYPE=Release -DSFML_DIR="</path/to/SMFL>"
cmake --build . --clean-first --target install -- -j 4

Windows install Errors

At this point if you are on windows you might run into errors. If the compilation succeeds but the installation fails, it is probably a permission problem.

To fix it, go to the directory CMake tried to install SDL into. In my case thats “C:\Program Files (x86)\TGUI”. If the directory doesn’t exist, create it. Right click on the directory and select Properties, then click on the Security tab and then click on Edit. Select the user that wants to install SDL, probably your current user, and then tick the Full Control box followed by Apply and Ok.

Rerun the installation command. This time it should work. I had one instance where it still couldn’t create/find the include directory. To fix that issue simply create it by hand inside the TGUI directory.

Consuming TGUI in your Projects

TGUI should now be installed on your system. Now it’s time to test if it works. For that create a simple CMake project. Inside the CMakeLists.txt file add the following lines of code.

find_package(SFML 2.5 COMPONENTS graphics audio REQUIRED)
find_package(TGUI 0.8 REQUIRED)

target_link_libraries(cool_project
                      PUBLIC sfml-graphics sfml-audio tgui)

Obviously change cool_project to the name of your executable. And add or remove the components you need/don’t need.

Now inside files which are part of the cool_project executable add the following lines of code.

#include <SFML/Graphics.hpp>
#include <TGUI/TGUI.hpp>

int main(){
  sf::RenderWindow window{{100, 100}, "SFML Window"};
  tgui::GUI gui{window};

  sf::Event event;
  while(window.isOpen()){
    while(window.pollEvent(event)){
      if (window_event.type == sf::Event::Closed)
        window.close();

      gui.handleEvent(event);
    }

    window.clear();
    gui.draw();
    window.display();
  }

  return 0;
}

cool_project should now compile.

More Windows Problems

Don’t forget the binaries

On Windows your program should compile now but crash when started. Theres a high chance that that is because it can’t find the TGUI runtime binaries. You’ll have to copy the TGUI binaries to one of two spaces. Possibility One is next to your executable, possibility two is into your windows system32 directory.
